# Meeting Transcriber

A fully local web app that transcribes recorded meeting audio on your Mac.
Nothing leaves your machine: transcription runs on the Apple Silicon GPU via
[mlx-whisper], speakers are identified with [pyannote.audio], and summaries are
generated by a local LLM through [Ollama].

- Accepts m4a, mp3, wav, aac, flac, ogg, and the audio track of mp4/mov
- Separates speakers, lets you listen to a snippet of each and assign names
- Exports a readable transcript (Markdown or TXT) with names and timestamps
- Generates a summary + action items with a local model, on demand

[mlx-whisper]: https://github.com/ml-explore/mlx-examples/tree/main/whisper
[pyannote.audio]: https://github.com/pyannote/pyannote-audio
[Ollama]: https://ollama.com

## Setup (one time)

```bash
brew install ffmpeg uv ollama
ollama pull qwen2.5:7b-instruct   # summary model (~4.7 GB)
uv sync
```

Speaker identification uses a gated (but free) HuggingFace model:

1. Create a free account at https://huggingface.co
2. Accept the conditions at https://huggingface.co/pyannote/speaker-diarization-community-1
3. Create a read token at https://huggingface.co/settings/tokens
4. `cp .env.example .env` and set `HF_TOKEN=hf_...`

The whisper and pyannote models download automatically on first use (~2 GB
total) and are cached locally; after that everything runs offline. Without an
`HF_TOKEN`, transcription still works — you just get a single unnamed speaker.

## Run

```bash
uv run python -m transcriber
```

Open http://localhost:8484, then:

1. Optionally fill in **Names & terms in this meeting** — attendee names,
   product names, jargon. Whisper uses it to bias spelling (it's passed as the
   `initial_prompt`), which is the biggest accuracy lever for proper nouns.
   The field persists in your browser between sessions.
2. Drop a recording onto the page. Progress shows each stage (a 1-hour meeting
   takes roughly 6–8 minutes with the full large-v3 model on an M-series GPU).
3. In **Speakers**, play each snippet and type the person's name — the whole
   transcript updates.
4. Click **Generate summary** for a local-LLM summary and action items
   (requires Ollama running: `ollama serve` or the Ollama app).
5. Export as Markdown or TXT.

Each job (audio + transcript JSON) is stored in `./data/<job-id>/`; the most
recent job is restored when you reopen the page.

## Testing

```bash
uv run pytest              # fast unit tests (no models)
uv run pytest -m slow -v   # full pipeline on a 3-speaker fixture; needs HF_TOKEN
```

The slow suite transcribes and diarizes `tests/fixtures/conversation.m4a` — a
synthesized 3-voice meeting with known ground truth — and asserts the correct
speaker count and ≥90% speaker-attribution accuracy. Regenerate the fixture
with `uv run python tests/fixtures/generate_conversation.py` (uses macOS
`say` voices).
